# [原创] TinyX86：一个纯 C 实现的轻量级 x86-32 模拟器，支持 FPU、PE 加载与交互式调试

## 0x00 前言

在逆向工程、算法还原或恶意代码分析中，我们经常需要对某段特定的汇编代码进行追踪或模拟执行。虽然市面上已有像 Unicorn 这样强大的引擎，但对于学习 x86 指令集架构、指令解码原理以及 PE 加载机制的同学来说，一个结构清晰、易于调试的轻量级模拟器往往是更好的切入点。

**TinyX86** 是我近期开发的一个实验性项目。它采用纯 C 语言编写，实现了完整的指令解码器、执行引擎、x87 FPU 以及一个交互式的类 OllyDbg/x64dbg 调试器。

---

## 0x01 项目特色：进程内模拟 (In-Process Emulation)

TinyX86 目前采用的是**宿主内存方案**，即模拟器与宿主进程共享同一地址空间。这种设计在逆向分析中非常实用：

1.  **算法还原 / 去混淆**：遇到 VMP 或其他混淆后的代码段，只需将其映射到内存，让 TinyX86 在同一进程内跑一遍，直接获取寄存器结果，性能极高。
2.  **API 零成本透传**：由于地址空间相同，模拟器在执行到 `CALL [IAT]` 时，可以直接跳转执行真实的系统 API（如 `MessageBox`），无需复杂的参数桥接。
3.  **辅助脱壳**：模拟执行壳的解密逻辑，直接利用宿主的 `GetProcAddress` 填充导入表。

## 0x02 核心模块架构

项目由四大核心模块组成：

| 模块 | 职责 | 实现文件 |
| :--- | :--- | :--- |
| **解码器 (Decoder)** | 从字节流中解析 x86 指令，填充解码上下文 | `disasm.c/h` |
| **执行引擎 (Executor)** | 根据解码结果更新 CPU 寄存器与 EFLAGS | `TinyX86.c/h` |
| **加载器 (Loader)** | 加载 PE 文件或原始二进制到模拟内存 | `PELoader.c/h` |
| **调试器 (Debugger)** | 提供单步、断点、内存/栈查看等交互功能 | `debugger.c/h` |

### 指令集支持情况
- **通用指令**：数据传输（MOV/PUSH/POP）、算术运算（ADD/SUB/MUL/DIV）、逻辑位移、分支跳转（Jcc/CALL/RET）。
- **串操作**：MOVS/STOS/LODS 等，支持 REP 前缀。
- **x87 FPU**：支持浮点栈操作、FLD/FST/FADD/FMUL 以及三角函数等。
- **系统指令**：NOP, HLT, INT 3 等。

## 0x03 交互式调试器演示

加载 PE 文件后，会自动进入类似命令行 GDB 的调试界面：

```text
=== TinyX86 Debugger Initialized ===
(0x00401000)  push ebp                        ; Next
(dbg) > r
--- CPU STATE ---------------------------------------------------
EAX: 0x00000000  EBX: 0x00000000  ECX: 0x00000000  EDX: 0x00000000
ESI: 0x00000000  EDI: 0x00000000  EBP: 0x00000000  ESP: 0x00410000
EIP: 0x00401000  CS:0000 DS:0000 ES:0000 SS:0000
EFLAGS: [ . . . . . . . ] (0x00000200)
--- FPU ---------------------------------------------------------
FPU SW: TOP=0  C3=0 C2=0 C1=0 C0=0
   Reg |      Double Value      | Logical ST(i)
   ----+------------------------+--------------
   R0  |           0.0000000000 | <- ST(0) TOP
   R1  |           0.0000000000 |    ST(1)
...
-----------------------------------------------------------------
```

**常用命令：**
- `s [N]`：单步步入 N 条指令。
- `n`：步过（跳过 CALL/LOOP 等）。
- `b <addr>`：设置十六进制地址断点。
- `m <addr>`：查看内存 Hex Dump。
- `k`：查看当前栈内容（自动标注 ESP/EBP）。
- `w <reg> <val>`：修改寄存器或标志位。

## 0x04 快速上手

### 编译
使用 Visual Studio 打开 `TinyX86.slnx` 即可编译。由于采用纯 C（配合部分 Windows API），迁移到其他平台也非常方便。

### 运行内置测试
运行程序后，在 Shell 输入 `t`：
```text
(Shell) > t
[Shell] Loading internal test...
--- [演示] 递归调用 power(2.0, 3) 计算过程 ---
(0x00400000)  push 0x3
(dbg) > c
Breakpoint hit at 0x00400011
(0x00400011)  int3
(dbg) > r
...
ST(0): 8.000000  TOP: 7  (结果正确：2^3 = 8)
```

## 0x05 未来计划 (Roadmap)

目前项目正处于活跃开发中，接下来的 Sprint 计划包括：
1. **完整的 IAT 修复与重定位**：支持任意 32 位 EXE 的加载运行。
2. **API Hook 拦截框架**：在模拟器层拦截系统调用，实现参数监控。
3. **SEH 异常处理**：支持 Windows 结构化异常处理，对抗反调试。
4. **污点分析底座**：为指令级追踪与自动化分析打下基础。

## 0x06 结语

TinyX86 的目标不是取代 QEMU 或 Unicorn，而是作为一个**透明、可控、易读**的国产 x86 模拟器实现范例。无论是想深入研究指令解码逻辑，还是需要一个轻量级的逆向辅助工具，希望它都能给你带来帮助。
